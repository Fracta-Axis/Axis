## Overview

MFSU-Crypt is built around three main components:

| Component | Description |
|-----------|-------------|
| **MFSU-KDF** | Memory-hard key derivation function based on fractal field dynamics |
| **MFSU Stream Cipher** | Keystream generator with key-dependent warmup steps |
| **HMAC-SHA3-256** | Message authentication using SHA3-based HMAC |

The test vectors in this repository allow:

1. Independent verification of each pipeline component **in isolation**
2. Regression testing for future versions of the reference implementation
3. Third-party cryptanalysis **without requiring a full system build**

### V7 — Warmup Steps (key-dependent)

**Input:** `dk = V1[0:64]`, `IV`

**Computation:**
```
h = SHA3-512(dk ‖ IV)
n_steps = 48 + (h[0] mod 64)
```

**Result:** `h[0] = 0x38 = 56₁₀` → `n_steps = 48 + 56 = 86` ✅
*(Valid range: 48–111)*

## Known Limitations

1. **No full encrypt/decrypt round-trip vector** — the current reference implementation uses `os.urandom` for salts and IVs. A deterministic encrypt vector is planned for **v1.1** of the reference implementation.

2. **V8 field norm at convergence** — shows `1.85` rather than the paper's `≈ 4.8` due to the short 10-step horizon. The bound holds; full convergence requires ~100 steps.

3. **NIST SP 800-22 vectors** — keystream statistical tests require ≥ 2 × 10⁶ bits and are not reproduced here. See **Table 6** of the main preprint.
