FractalShield

A core-agnostic framework for oracle-free layered encryption with geometric cost escalation. Any function satisfying the four laws below inherits the full security proofs.
Miguel Angel Franco León Fracta-Axis Project github.com/Fracta-Axis/Fractalyx
Abstract

FractalShield simultaneously achieves oracle-free verification (OFV) — the attacker cannot determine password correctness without spending the full key-derivation cost per layer — geometric cost escalation (3.5×, 7.5×, 15.5× baseline per level), statistical indistinguishability of real versus decoy ciphertext layers, and serverless offline operation.

The framework is intentionally core-agnostic. Any function satisfying Definition 3.1 (C1–C4) can serve as the cryptographic core without altering the four security theorems.

§ 3.1 The Four Laws — Definition 3.1

A function F : {0,1}^κ × {0,1}^128 × ℕ → {0,1}* is a valid FractalShield core if and only if it satisfies all four properties below. Any such F inherits Theorems 5.1, 5.2, 5.5, and 5.6 without modification.

C1 INPUT SENSITIVITY
Avalanche Property
∀ k ≠ k′: diffBits(F(k,IV,L), F(k′,IV,L)) ≈ 50%
A single-bit change in the key must flip approximately half of the output bits. This ensures that no partial information about the key leaks from comparing ciphertexts. Without this, an adversary could do hill-climbing over keyspace.
C2 OUTPUT PSEUDORANDOMNESS
PRG Property
F(k, IV, ·) ≈_c U_L for all PPT distinguishers
The output must be computationally indistinguishable from uniform random. This is the foundation for layer statistical indistinguishability — an attacker cannot tell which layer holds the real plaintext by analyzing ciphertext bytes.
C3 COST CONTROLLABILITY
Parameterised Cost
cost(F(k, IV, L; M)) is monotone increasing in M
The evaluation cost must be tunable via a parameter M. The framework uses this to assign cost M₀ · 2ⁱ to layer i, forcing the attacker to pay the geometric sum 2^N − 1 per password guess.
C4 MEMORY HARDNESS
Ω(M) RAM Requirement
RAM(F at param M) = Ω(M)
Evaluating F must require memory proportional to M, bounding parallel GPU attacks. A device with G bytes of RAM can run at most ⌊G/M⌋ threads simultaneously — this is what makes brute-force physically expensive at scale.

§ key insight The asymmetry principle

The core architectural idea: the legitimate user decrypts exactly one layer. The attacker must decrypt all N layers at escalating cost because they cannot know which layer holds the real plaintext (the layer order is encrypted under the real key).

§ inheritance What every valid core inherits

If your function F satisfies C1–C4, all four theorems apply immediately, regardless of the internal structure of F.

Thm 5.1
Integrity
Any PPT adversary producing a modified ciphertext that passes MAC has negl(κ) advantage. Basis: SUF-CMA of HMAC-SHA3-256.
Thm 5.2
IV Uniqueness
Two encryptions under the same password reuse an IV with probability ≤ n²/2¹²⁸. Basis: birthday bound.
Thm 5.5
Oracle-Free Verification (OFV)
No PPT adversary can determine password correctness without spending C_attacker(ℓ) per attempt. Requires Thm 5.1 + C2 + false-positive bound 5/2⁴⁰.
Thm 5.6
IND-CCA2 (under ROM)
Adv_IND-CCA2(A) ≤ 2q²/2²⁵⁶ + 5/2⁴⁰. Four-game hybrid proof. Standard-model reduction is Open Problem 4.

§ 10 Open problems
OP1
Analytical injectivity of G. Theorem 6.5 is conditional on empirical Lemma 6.2. Rigorous algebraic proof of injectivity needed. Suggested: Schwartz-Zippel applied to the polynomial approximation of the cubic nonlinearity.
OP2
DAG memory-hardness tight bound. Phase 2 scratchpad coverage is 20–53% empirically. Proof of Ω(4 MB) unconditionally under the parallel ROM is needed.
OP3
Replay-attack mitigation. Valid ciphertexts from other sessions are currently accepted. Fix: include session-context identifier in MAC derivation. Does not affect any threat model in this paper.
OP4
IND-CCA2 without ROM. Theorem 5.6 assumes the Random Oracle Model. Standard-model reduction is desirable for high-assurance deployments.
